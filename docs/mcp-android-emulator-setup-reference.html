MCP + Android Emulator Setup Reference

This is the practical handoff for reproducing the Android native-app MCP setup used in this repo on Windows. It combines the repo MCP config, the native-app docs, and the exact fixes needed to make the session work on this machine.

Repo root: C:\Studies\Agentic AI\playwright-ui-framework

APK used: C:\Studies\Agentic AI\playwright-ui-framework\Mobile_apps\Android.SauceLabs.Mobile.Sample.app.2.7.1.apk

Working emulator: Pixel 5, emulator-5554, Android 11

1. What This Setup Does

  1. Android Studio provides the Android SDK and emulator.
  2. Appium provides native Android automation.
  3. The uiautomator2 Appium driver drives the app on the emulator.
  4. MCP uses the repo-level .mcp.json file to talk to Appium.
  5. A PowerShell helper script starts Appium with the Windows environment variables it needs.

2. Files In This Repo That Matter

.mcp.json

{
  "mcpServers": {
    "wdio-mcp": {
      "command": "npx",
      "args": ["-y", "@wdio/mcp"],
      "env": {
        "APPIUM_URL": "127.0.0.1",
        "APPIUM_URL_PORT": "4723",
        "APPIUM_PATH": "/"
      }
    }
  }
}

This tells MCP to use @wdio/mcp and connect to Appium at http://127.0.0.1:4723/.

Helper script

The repo also contains scripts/create-native-android-session.ps1. It kills old Appium processes, sets SDK and Java environment variables, starts Appium, and creates the working Android session.

3. Software Needed On A New Windows Computer

  1. Install Node.js.
  2. Install Android Studio.
  3. Install Appium globally.
  4. Install the Appium uiautomator2 driver.
  5. Clone this repo and install the repo dependencies.
npm install -g appium
appium driver install uiautomator2

If PowerShell blocks appium.ps1, use appium.cmd instead. On this machine that path was C:\Users\aqadi\AppData\Roaming\npm\appium.cmd.

4. Android Studio And Emulator Setup

  1. Install Android Studio.
  2. Install the Android SDK.
  3. Make sure platform-tools, build-tools, and an Android 11 system image are installed.
  4. Create an AVD in Device Manager.
  5. Start the emulator before asking the AI to create the session.

The working emulator here was effectively a Pixel 5 profile on Android 11, visible to ADB as emulator-5554.

adb devices

If adb is not on PATH, call it with its full SDK path. On this machine it was C:\Users\aqadi\AppData\Local\Android\Sdk\platform-tools\adb.exe.

5. Windows-Specific Fixes We Needed

ADB not on PATH

Use the full path to adb.exe, or add the SDK platform-tools folder to PATH.

PowerShell blocked Appium

Use appium.cmd instead of appium if appium.ps1 is blocked by execution policy.

Appium could not find Android SDK

$env:ANDROID_HOME="C:\Users\aqadi\AppData\Local\Android\Sdk"
$env:ANDROID_SDK_ROOT="C:\Users\aqadi\AppData\Local\Android\Sdk"

Appium could not find Java

$env:JAVA_HOME="C:\Program Files\Android\Android Studio\jbr"
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"

Important activity fix: the APK initially looked like it would launch through a splash activity, but the stable activity was com.swaglabsmobileapp.MainActivity. The working session needed appPackage, appActivity, and appWaitActivity all aligned to that app.

7. Recommended Startup Flow

  1. Start the Android emulator.
  2. Run adb devices.
  3. Make sure .mcp.json is present in the repo.
  4. Start Appium.
  5. If Windows needs it, use the helper script instead of launching Appium manually.
  6. Reload the MCP client so it picks up the repo config.
  7. Ask the AI to start the native Android session.
  8. Continue the test flow from the visible app screen.

8. How We Verified Things During Setup

adb devices
appium.cmd -v
appium.cmd driver list --installed
Invoke-RestMethod -Uri "http://127.0.0.1:4723/status" -Method Get

The most useful log files were appium-native-session.log and appium-native-session.err.log. They showed missing SDK variables, missing Java, wrong activity names, and session creation failures very quickly.

11. What To Change On Another Computer

  1. Android SDK path
  2. Java path
  3. Global npm path to appium.cmd
  4. Emulator name or UDID
  5. APK path

The MCP config itself can stay the same as long as Appium still runs locally at 127.0.0.1:4723.

12. Troubleshooting Checklist

  1. Make sure the emulator is booted.
  2. Run adb devices.
  3. Confirm uiautomator2 is installed.
  4. Confirm Appium is listening on port 4723.
  5. Confirm ANDROID_HOME and ANDROID_SDK_ROOT are set.
  6. Confirm JAVA_HOME points to a valid Java runtime.
  7. Check the Appium log for the exact failing step.
  8. If the app launches but Appium times out, confirm the real activity and update appWaitActivity.
